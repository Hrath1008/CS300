# CS300
CS 300 Module 8 Journal

•	What was the problem you were solving in the projects for this course?
The goal of these projects was to develop software for ABC University (ABCU) that helps computer science advisors access course information for students. The program would allow the user to load a list of courses from a file, print out the sorted list of courses, and search for a specific course within that list and see the course’s name, number, and prerequisites. 
Project One was specifically for developing pseudocode for the software and figuring out which data structure (vector, hash table, or binary search tree) would be the best suited for this problem based on its runtime. Project Two was specifically for developing the actual program using one of the three data structures.
•	How did you approach the problem? Consider why data structures are important to understand.
I approached this problem by first thinking about the general requirements for this software, which was to handle a file, sort the data, and search for objects. I then expanded on this, by developing pseudocode for the three data structures and performing a Big O analysis to find which data structure would suit the needs of the user and do so quickly and efficiently. 
•	How did you overcome any roadblocks you encountered while going through the activities or project?
I was faced with a few roadblocks when developing the code for this project. I had the most trouble with validating the data from the file and sorting the data. To overcome these roadblocks, I referenced the Zybooks and looked up many tutorials and guides online. 
•	How has your work on this project expanded your approach to designing software and developing programs?
Working through this project, I have discovered how helpful it can be to write out pseudocode and develop a plan of action for designing software and developing programs. I can figure out the best possible solution to a problem by workshopping different ways to approach the problem beforehand which can save me time and avoid errors and having to rewrite code when it does not work. 
•	How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Working through this project has helped me understand that there are multiple ways of developing a program. Not every method works for every program, you need to find the best method that will suit the needs of the program. I have found that developing a plan also can help with making sure that my code is organized and readable, which in turn can make it maintainable and easily adapted when needed.
